# GitHub 仓库操作技能

## 任务目标
在 GitHub 网站上执行各种操作，包括搜索、导航、star、fork、查看内容等。

## 工作流程

### 1. 导航到 GitHub
- 打开 Safari 浏览器
- 访问 github.com
- 确认页面加载完成

### 2. 搜索仓库
- 点击顶部搜索框
- 输入搜索关键词（如 "turix-cua"）
- 按 Enter 或点击搜索按钮
- 等待搜索结果加载

### 3. 访问仓库页面
- 在搜索结果中找到目标仓库
- 点击仓库名称进入仓库主页
- 确认页面元素加载完成（README、文件列表等）

### 4. 执行操作

#### Star 仓库
- 找到右上角 Star 按钮
- 检查当前状态（已 star/未 star）
- 如未 star，点击 Star 按钮
- 确认按钮状态变化

#### Fork 仓库
- 找到右上角 Fork 按钮
- 点击 Fork 按钮
- 等待 Fork 完成（出现进度提示）
- 确认跳转到 fork 的仓库页面

#### 查看 Issues
- 点击顶部导航栏 "Issues" 标签
- 浏览 issue 列表
- 可点击具体 issue 查看详情

#### 查看 Code
- 点击顶部导航栏 "Code" 标签
- 浏览文件结构
- 可点击文件查看内容

#### 查看 Pull Requests
- 点击顶部导航栏 "Pull requests" 标签
- 浏览 PR 列表
- 可点击具体 PR 查看详情

### 5. 收集信息（如需要）
- 记录仓库信息：
  - 仓库名称
  - Star 数量
  - Fork 数量
  - 主要编程语言
  - 最新更新时间
  - 描述信息

## 注意事项

### 登录状态
- 检查是否已登录 GitHub
- 如需登录，询问用户提供凭据
- 不要自动输入密码（除非用户明确要求并提供）

### 操作确认
- Star 前确认按钮状态
- Fork 前确认是目标仓库
- 不要重复 star（先检查状态）

### 速率限制
- 避免短时间内大量操作
- 如遇速率限制，等待后继续
- 不要自动化敏感操作（如删除）

### 安全边界
- 只访问公开仓库（除非已登录且有权限）
- 不修改仓库内容（除非明确要求）
- 不执行删除操作
- 不提交代码（除非明确要求）

## 错误处理

### 常见问题
- **搜索无结果**: 尝试不同关键词或检查拼写
- **页面加载失败**: 刷新页面或检查网络
- **需要登录**: 提示用户登录
- **404 错误**: 确认仓库名称正确或仓库存在

### 恢复策略
- 返回上一页重试
- 重新搜索
- 使用 GitHub 首页导航
- 直接访问已知 URL

## 预计步骤
- 简单任务（搜索 +star）：5-8 步
- 中等任务（收集信息）：10-15 步
- 复杂任务（多仓库操作）：20-30 步

- [ ] 成功访问目标仓库
- [ ] 完成指定操作（star/fork/查看等）
- [ ] 确认操作成功（按钮状态变化等）
- [ ] 如需要，记录并返回信息
