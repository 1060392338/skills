# 项目知识分层存储策略

## 触发条件
当用户提到一个新项目、或在会话中确认某个项目已建立稳定结构时，使用此策略。

## 原则

```
记忆 (Memory)   → 简短指针（≈100字）指向 Skill
Skill           → 完整项目知识（架构/运行/调试/隔离）
会话历史搜索     → 追溯具体过程和问题解决记录
```

## 步骤

### 1. 识别"项目知识"范围
- 项目目录、运行方式、架构图 → Skill
- 调试命令、常见错误、隔离方案 → Skill
- 扩展方式、API细节、参数说明 → Skill
- 用户偏好、环境特有问题、约定俗成的做法 → 可酌情留记忆

### 2. 压缩记忆
- 删除所有项目详细描述条目
- 新增一条紧凑指针，格式：
  ```
  「{项目名}」目录：{Path}。详细运行方式、架构、调试见 skill: {skill-name}。
  ```

### 3. 写入 Skill
- 完整SKILL.md包含：
  - 项目信息（目录/Python版本/依赖）
  - 架构概览（目录树或模块图）
  - 运行方式（命令示例）
  - 核心设计（Agent/隔离/扩展）
  - 调试要点
  - 常见错误

### 4. 后续使用
- 当用户提到项目名 → 自动加载对应 Skill
- 遇到新问题/修复 → 更新 Skill 的调试/常见错误部分
- 会话历史不截断 → 用户可继续当前会话深入

## 示例

### 记忆条目（轻量）
```
「头条多Agent自动化」项目目录：~/.hermes/ai-news-workflow/。详细运行方式、架构、调试见 skill: toutiao-news-workflow。
```

### Skill 结构
参考 `toutiao-news-workflow` Skill，包含架构概览、运行方式、4个Agent角色、工作流、多租户隔离、发布器机制、调试命令、常见错误。
