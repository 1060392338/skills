### 已知问题

1. **LLM幻觉** — 子进程超时时LLM可能脑补"全部认领成功"。修复：工具返回 `[RESULT]` 结构化标记，LLM严格按 `success=true/false` 判断。
2. **Windows中文编码** — argparse传中文店名会GBK截断。修复：state文件的`claim_store`置空。
3. **采集箱虚拟滚动** — 只渲染8-9行，勾选必须跨页遍历。已在 `run_compliance_claim.py` 和 `run_publish.py` 修复为跨页勾选。
4. **发布到发布中即算成功** — ERP异步发布(约1件/分钟)，不等完成。
